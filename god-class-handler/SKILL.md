# God Class Handler

将本技能用于“超大类/超大方法”的可控重构，优先保证行为不变和调用迁移完整。

## 输入要求

开始前收集以下信息：

1. 目标文件路径与语言（Java/Kotlin/TypeScript/C#/Python 等）。
2. 目标类名、相关接口/父类、关键依赖。
3. 构建与测试命令（至少一个可执行校验命令）。
4. 允许的重构边界（是否允许改公共 API、是否允许新增文件）。

若输入不完整，先做静态扫描并给出缺失项，再继续最小可行重构。

## 规模分流规则

将“类总行数 + 核心方法行数”一起评估，采用更严格的路径：

1. `<=500` 行  
   重点做“超大方法缩小 + 功能拆分”，通常由主 agent 直接完成。
2. `500-1000` 行  
   先划分功能模块，再分批迁移到模块类/服务类，保留兼容入口。
3. `1000+` 行  
   使用主 agent + 子 agent 编排。子 agent 仅处理单一功能单元；主 agent 负责合并、删除、清理与全量调用校验。

## 执行流程

### 步骤 1：盘点与切分（主 agent）

1. 枚举目标类中的：
   - 字段/常量
   - 公共方法
   - 私有方法
   - 外部调用点
2. 生成“功能单元图”（每个单元保持单一职责）：
   - 参数解析/校验
   - 业务编排
   - 持久化访问
   - 外部 API/基础设施调用
   - 结果组装/格式化
3. 对每个单元标记：
   - 依赖字段/常量
   - 被哪些方法调用
   - 是否可独立迁移

### 步骤 2：按阈值选择重构策略

#### A. `<=500` 行

1. 拆分超大方法为语义化私有方法（提取 early-return、校验、分支处理）。
2. 将重复逻辑提取为 helper/utility（仅在同一语义域内）。
3. 保持外部方法签名不变，优先做内部整理。

#### B. `500-1000` 行

1. 先定义模块边界（示例：`ValidationModule`、`ExecutionModule`、`MapperModule`）。
2. 逐模块迁移方法和常量，避免一次性大搬迁。
3. 原类保留薄编排层，直到调用点迁移稳定后再收缩。

#### C. `1000+` 行

1. 主 agent 先产出“子任务清单”，每个子任务只含一个功能单元。
2. 将子任务委托给子 agent 并行/串行执行（根据依赖关系）。
3. 每个子 agent 输出：
   - 新增/修改文件
   - 迁移的方法列表
   - 调用点替换列表
   - 风险与待主 agent 处理项
4. 主 agent 合并后做最终收尾：
   - 从原类删除已迁移方法
   - 删除原类中不再使用的常量/字段
   - 清理无用 import/依赖
   - 执行全量调用检查与回归测试

## 子 agent 委托规范

对子 agent 的任务描述必须包含：

1. 单一功能单元范围（禁止跨单元扩展）。
2. 明确输入/输出契约（方法签名、异常语义、返回值语义）。
3. 必做清单：
   - 迁移后保持行为一致
   - 补齐该单元全部调用点
   - 更新或新增最小必要测试
4. 禁止事项：
   - 修改其他单元逻辑
   - 无依据地变更公共 API
   - 跳过编译/测试验证

可复用模板见 [references/subagent-task-template.md](references/subagent-task-template.md)。

## 主 agent 收尾清单（必须执行）

1. 删除原类中“已迁移方法”的旧实现（禁止保留失效重复逻辑）。
2. 删除原类未再引用的常量、字段、私有工具方法。
3. 使用全文检索确认调用迁移无遗漏（方法名、类名、旧常量名）。
4. 执行构建与测试；至少覆盖：
   - 编译通过
   - 受影响模块测试通过
   - 关键路径 smoke test 通过
5. 输出迁移矩阵：
   - 旧方法 -> 新归属
   - 旧调用点 -> 新调用点
   - 删除项（方法/常量/字段）

## 防漏调用检查（强制）

对每个已迁移方法执行以下检查：

1. 迁移前记录调用点基线（`rg "<methodName>\\("`）。
2. 迁移后再次检索，确认旧调用已清零或仅保留兼容桥接。
3. 若存在反射/字符串调用，增加额外搜索关键字与运行时验证。
4. 对外暴露 API 若改名，必须提供迁移层或一次性全量替换并测试。

## 质量门禁

任一项失败则不得结束：

1. 编译失败。
2. 关键测试失败。
3. 发现遗漏调用点。
4. 原类仍保留已迁移且不再使用的重复实现。
5. 原类中仍存在未使用常量且可安全删除。

## 输出格式（每次执行后）

1. 重构策略：`<=500` / `500-1000` / `1000+`
2. 模块划分与子任务列表
3. 已完成迁移方法清单
4. 已删除方法/常量/字段清单
5. 调用点迁移校验结果
6. 测试与构建结果
7. 剩余风险与下一步
